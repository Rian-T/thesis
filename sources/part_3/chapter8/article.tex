\section{Introduction}

A fixed classifier can only return the classes it saw during training. Clinical
study forms do not work that way. The fields of a form change from one cohort to
the next, and a model trained on one set of fields cannot answer a question about
a field it has never seen. This chapter builds an extractor that reads clinical
text and returns the values of fields that were not fixed in advance, without
using any hospital label to learn them.

The idea is to describe each requested type in natural language at the moment the
model is used, rather than to fix the set of types when the model is trained. The
output interface can then accept a new field without replacing its classification
layer. GLiNER~\citep{zaratiana2024gliner} follows this design: it represents
candidate spans and type descriptions in a shared space, then selects the spans
that match the requested description (\Cref{fig:p3-open-query}). This interface
makes new queries possible, but useful extraction still depends on how well the
model represents medical descriptions.

\section{The Open-Vocabulary Extractor}

Two parts have to work together for description-based extraction. The model needs
a space in which medical descriptions and text spans can be compared, and it needs
to be trained to select spans from that space. We build the first part on top of a
biomedical encoder, and the second part on generalist annotations that carry no
hospital label.

MedEmbed builds the first part, the space in which descriptions and spans are
compared. It starts from ModernCamemBERT-bio, the biomedical encoder developed in
\Cref{chap:objectives}. Medical terms and relations derived from the resources
of \Cref{chap:ontobook} provide the contrastive pairs that train this description
space, as summarized in \Cref{fig:p3-medembed}.

\input{sources/part_3/figures/fig07_medembed}

We first test this space through terminology retrieval. OntoBench-FR asks a
model to match a French clinical term to its correct entry across four
terminologies, CIM-10, ATC, CCAM, and ICD-O. MedEmbed reaches 60.35\% mean
accuracy at rank one, compared with 50.84\% for the strongest baseline
(\Cref{tab:p3-medembed}).

The second part is the extractor that selects spans. It follows the GLiNER2
span--description design \citep{zaratiana2025gliner2}, trained on the generalist
annotations from \Cref{chap:synthetic} using equal proportions of descriptions
produced by the annotator and their canonical forms, with no training label from
a hospital record. The released checkpoint has 150 million parameters and is named
ModernCamemBERT-bio-gliner-base. We write its full model name as
ModernCamemBERT-bio-gliner and use the short form MC-bio-gliner in the rest of
this part.

This training produces a generalist model that can query types through their
descriptions. The model can later be adapted to the lymphoma form using its 89
field descriptions and generated records. Performance on that form and transfer
to other descriptions remain separate questions, since specializing the model
does not guarantee that its initial openness is preserved.


\section{Public Clinical Text in the Training Corpus}

Does public clinical text actually help this extractor? We answer with an
ablation on the composition of the generalist corpus, keeping the evaluation set
fixed to the 200 records shared by all runs. The complete corpus is compared with
a corpus without public clinical paragraphs and with a second biomedical control
corpus. \Cref{fig:p3-section-supervision} reports the global effect and breaks it
down by form section. All scores in this section are micro-averaged value-$F_1$,
the metric defined in \Cref{chap:evaluation}.

\begin{figure}[t]
  \centering
  \includegraphics[width=0.86\textwidth]{plots/part_3/output/fig08_section_supervision.pdf}
  \caption{Value-$F_1$ by corpus composition: the global effect, then broken down
  by eCRF section; the biomedical control is reported only at the global level.}
  \label{fig:p3-section-supervision}
\end{figure}

This ablation measures the open-vocabulary extractor before any task adaptation.
It is applied zero-shot to a 200-record probe, so its absolute value-$F_1$ sits
far below the task-adapted model of \Cref{chap:evaluation}, and only the
differences between corpus conditions carry meaning here. The global effect is
modest: value-$F_1$ reaches 0.205 with the complete corpus, compared with 0.183
without public clinical paragraphs and 0.170 for the biomedical control. The
difference between the first two conditions is 0.022.

The aggregate result hides a more concentrated effect. Removing public clinical
text reduces first-line treatment value-$F_1$ from 0.211 to zero and second-line
treatment value-$F_1$ from 0.050 to zero, while the larger disease-history
section changes from 0.242 to 0.228. Public clinical text therefore has its
largest observed effect on the treatment sections of these 200 generated records.


\section{Transfer to Unseen Descriptions}

OntoBench-FR tests terminology retrieval, but it does not show whether a new
description helps locate spans in text. No manual span reference is available for
arbitrary descriptions, so we evaluate with a reference-free LLM judge. We pool
and deduplicate the candidate spans from the compared models, remove their model
identity, and ask the judge whether each span matches the requested description.
Mean average precision (MAP) then evaluates the resulting ranking without choosing
a decision threshold, a score we call LLM-judge MAP.

We call descriptions constructed to be absent from the evaluation benchmarks
\emph{benchmark-absent}, though they are not guaranteed to be absent from every
training source, and we report them separately from familiar descriptions. The
comparison changes only the description encoder, replacing ModernCamemBERT-bio
with MedEmbed while the extractor remains fixed. \Cref{fig:p3-novel-types} reports
both conditions.

\begin{center}
  \includegraphics[width=0.78\textwidth]{plots/part_3/output/fig09_novel_types.pdf}
  \captionof{figure}{LLM-judge MAP for familiar and benchmark-absent type descriptions.}
  \label{fig:p3-novel-types}
\end{center}

For familiar descriptions, LLM-judge MAP increases from 0.478 with
ModernCamemBERT-bio to 0.517 with MedEmbed, a gain of 0.039. For benchmark-absent
descriptions, it increases from 0.333 to 0.382, a gain of 0.049. The larger gain
in the second condition is consistent with the intended role of contrastive
description training. These scores reflect the preferences of a reference-free LLM
judge, not agreement with a human medical reference.


\section{Limitations}

The same model family contributes to corpus annotation, record generation, and
the evaluation of benchmark-absent descriptions, which may favour shared
conventions. The public-text ablation changes an entire source family and does
not isolate one linguistic property. Its section-level differences describe the
effect of that intervention, but they do not identify its linguistic cause.


\section*{Conclusion}

Trained only on generalist annotations, with no hospital label, the extractor
still reads clinical text and returns the values of fields described in natural
language at inference time. What makes its open queries useful is the description
encoder. MedEmbed improves terminology retrieval on OntoBench-FR and helps locate
spans for benchmark-absent descriptions, so the extractor can be asked for new
fields rather than only the ones fixed at training time. Two
questions stay open. Target-form performance and transfer to other descriptions
must still be measured separately, and local span--description similarity does
not determine the role of a repeated mention in a patient trajectory.
\Cref{chap:evaluation} therefore moves from local extraction to complete
longitudinal form filling.
